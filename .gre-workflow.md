# Gre 工作规范

## 主动汇报原则

### 1. 每完成一个 US 自动汇报
- 立即告知完成状态
- 简要说明实现的功能
- 列出关键文件

### 2. 遇到阻塞主动说明
- 错误类型（429 quota / 代码错误 / 依赖缺失）
- 尝试的解决方案
- 需要用户决策的选项

### 3. 定期心跳
- 每 30 分钟无进展时主动告知状态
- "仍在推进中，当前在做..."
- 预计完成时间

### 4. 长时间阻塞的兜底方案
- 使用 cron 定时任务唤醒
- 即使 quota 超限阻塞，也能通过定时任务恢复

## 汇报模板

### 完成汇报
```
✅ US-XXX 完成！
文件：xxx.py
核心功能：...
下一步：...
```

### 阻塞汇报
```
⚠️ 遇到阻塞：
类型：429 quota 超限
已尝试：等待 60s 后重试
选项：
1. 继续等待 quota 恢复
2. 改用其他方案
3. 暂停，明天继续
```

### 心跳汇报
```
💓 状态更新：
已运行：X 分钟
当前：正在实现 US-XXX
进度：70%
预计：还需 Y 分钟
```
